## 七、常用运维操作手册

> ⭐ **重点掌握：日常运维90%的操作都在这里**

### 7.1 服务启停与状态查看

| 操作类型 | Nginx | MariaDB | PHP-FPM |
|----------|-------|---------|---------|
| 启动服务 | `systemctl start nginx` | `systemctl start mariadb` | `systemctl start php-fpm` |
| 停止服务 | `systemctl stop nginx` | `systemctl stop mariadb` | `systemctl stop php-fpm` |
| 重启服务 | `systemctl restart nginx` | `systemctl restart mariadb` | `systemctl restart php-fpm` |
| 查看状态 | `systemctl status nginx` | `systemctl status mariadb` | `systemctl status php-fpm` |

**开机自启相关命令：**

```bash
# 设置开机自启
systemctl enable 服务名

# 取消开机自启
systemctl disable 服务名

# 查看是否开机自启
systemctl is-enabled 服务名
```

---

### 7.2 Nginx配置修改与重载

**标准操作流程（必须遵守！）：**

1. **先备份原配置文件**
```bash
cp /etc/nginx/conf.d/default.conf /etc/nginx/conf.d/default.conf.bak
```

2. **修改配置文件**
```bash
vim /etc/nginx/conf.d/default.conf
```

3. **语法校验（必须做！）**
```bash
nginx -t
```

4. **平滑重载（不中断服务）**
```bash
nginx -s reload
```

> ⚠️ **重要提醒：修改配置后一定要先 `nginx -t` 校验，确认无误再 reload，否则可能导致Nginx启动失败！**

---

### 7.3 日志查看与排错

| 日志类型 | 日志路径 | 常用查看命令 |
|----------|----------|--------------|
| Nginx访问日志 | `/var/log/nginx/access.log` | `tail -f /var/log/nginx/access.log` |
| Nginx错误日志 | `/var/log/nginx/error.log` | `tail -f /var/log/nginx/error.log` |
| PHP-FPM错误日志 | `/var/log/php-fpm/www-error.log` | `tail -f /var/log/php-fpm/www-error.log` |
| MariaDB日志 | `/var/log/mariadb/mariadb.log` | `tail -f /var/log/mariadb/mariadb.log` |

**常用日志查看技巧：**

```bash
# 实时查看日志（最常用）
tail -f 日志文件路径

# 查看最后20行

# 搜索包含关键词的行

```

### 7.4 端口与进程查看

```bash
# 查看所有监听端口
```

### 7.5 文件与目录权限操作

**修改所有者：**

```bash
# 递归修改目录所有者
chown -R nginx:nginx /usr/share/nginx/html/
```

**修改权限：**

```bash
# 目录权限755，文件权限644（标准配置）
find /usr/share/nginx/html/ -type d -exec chmod 755 {} \;
find /usr/share/nginx/html/ -type f -exec chmod 644 {} \;
```
