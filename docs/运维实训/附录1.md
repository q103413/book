### 第 8 页：典型问题与解决方案（加分核心页）

**内容要点**

列出 2-3 个实操中印象最深的问题，每个问题包含三要素：

1. 故障现象：具体的报错信息或异常表现
2. 原因分析：定位问题的思路，根因是什么
3. 解决方法：最终的解决步骤与验证结果

**参考示例**

- 问题 1：`yum install nginx`提示`No package nginx available`

  原因：CentOS 默认 base 源不包含 Nginx 安装包，缺少扩展源

  解决：安装 epel 扩展源，或配置 Nginx 官方 yum 源

- 问题 2：访问 PHP 页面显示 502 Bad Gateway

  原因：PHP-FPM 服务未启动，或 Nginx 与 PHP-FPM 通信地址不匹配

  解决：重启 PHP-FPM 服务，检查 Nginx 配置的 fastcgi_pass 参数并重载

- 问题 3：上传主题插件提示权限不足 / 需要 FTP 凭证

  原因：网站目录所有者与 PHP 运行用户不一致，无写入权限

  解决：递归修改网站目录所有者为 nginx 用户

**制作提示**：这是最能体现实操能力与排错思维的页面，一定要认真做，答辩时重点讲解。

## 二、PPT制作（120分钟）

学生分组制作PPT，教师巡回指导。

**技术要求**：

- 建议使用 PowerPoint / WPS / 腾讯文档
- 截图需清晰，标注关键信息
- 代码块需格式化展示



### PPT制作注意事项

- **简洁明了**：每页不要太多文字，要点式呈现，字要大一点，后排能看清

- **图文并茂**：多放截图、架构图，少放大段文字和大段代码

- **逻辑清晰**：按"为什么做→怎么做→做成了啥→学到了啥"的逻辑线

- **重点突出**：把你们组做得好的地方、有特色的地方突出展示

- **统一风格**：字体、颜色、排版风格统一，不要花里胡哨

- **不要念稿子**：PPT是提示用的，不是照着念的，答辩时要讲出来



### 答辩技巧

1. **控制时间**：每组展示5分钟，提问3分钟，提前演练，不要超时

2. **自信大方**：声音洪亮，看着观众，不要低着头念PPT

3. **讲思路，不讲细节**：不要每个命令都念一遍，讲整体思路、关键步骤、遇到的问题

4. **诚实面对不会的问题**：不会就说"这个问题我们之前没考虑到，下来会研究一下"，不要瞎编

5. **分工配合**：可以一个人主讲，其他人补充；也可以每人讲一部分，衔接好

6. **提前准备**：提前到现场测试电脑、投影、网络，避免临场出问题

### Linux目录结构详解

| 目录    | 作用说明                                                     |
| ------- | ------------------------------------------------------------ |
| `/`     | **根目录**，所有目录的起点，整个文件系统的最顶层             |
| `/root` | root超级用户的家目录，普通用户进不去                         |
| `/home` | 普通用户的家目录，每个用户在这有一个子目录，比如/home/zhangsan |
| `/etc`  | **配置文件目录**，系统和服务的配置文件都在这里，运维最常逛的目录 |
| `/usr`  | 系统软件资源目录，安装的软件大都在/usr/bin、/usr/sbin、/usr/local下 |
| `/var`  | **可变数据目录**，日志\(/var/log\)、数据库文件、网站根目录等经常变化的文件 |
| `/tmp`  | 临时文件目录，所有人都能写，重启可能会清空                   |
| `/dev`  | 设备文件目录，硬件设备都抽象成文件放在这里，比如/dev/sda是第一块硬盘 |
| `/proc` | 虚拟文件系统，存放当前内核和进程的信息，不是真实的文件，存在内存里 |
| `/bin`  | 基础命令目录，存放系统启动和运行必需的命令，比如ls、cp       |
| `/sbin` | 超级用户命令目录，只有root能用的系统管理命令                 |



常见问题FAQ

### 云服务器相关

1. **Q：云服务器太贵了，有没有免费的学习方式？**
   A：①阿里云、腾讯云新用户都有免费试用；②学生有学生特惠；③可以用VirtualBox/VMware在本地装虚拟机学习，就是没有公网IP。  

2. **Q：按量付费和包年包月选哪个？**
   A：短期学习用量付费，用完就释放，几毛钱一天。长期用包年包月更划算。  

3. **Q：服务器密码忘了怎么办？**
   A：阿里云控制台可以重置密码，然后重启实例就生效了。  

4. **Q：服务器被入侵了怎么办？**
   A：学习用的服务器直接重置系统盘最快。生产环境就要排查入侵路径、修复漏洞、备份数据了。  



### Linux操作相关

1. **Q：命令记不住怎么办？**
   A：不用死记，用多了自然就记住了。记不住的时候man命令看帮助，或者搜一下。常用的也就那十几个。  

2. **Q：rm \-rf / 真的会删光所有东西吗？**
   A：是的，千万不要试！生产环境执行这个就是重大事故。删除前一定要用ls确认路径，重要文件先备份。  

3. **Q：vi编辑器怎么用？不会退出怎么办？**
   A：按Esc进入命令模式，输入 :q\! 强制退出不保存，:wq 保存退出。新手可以先用nano编辑器，更简单直观。  

4. **Q：权限755、644是什么意思？怎么算的？**
   A：三个数字分别代表所有者、所属组、其他人的权限。r=4，w=2，x=1。7=4\+2\+1=rwx，5=4\+1=r\-x，6=4\+2=rw\-。所以755就是所有者读写执行，组和其他人读执行。  



### 部署相关

1. **Q：浏览器访问IP打不开网站？**
   A：按顺序排查：①ping一下IP通不通；②安全组开80端口了吗；③防火墙关了吗；④Nginx启动了吗；⑤Nginx配置对不对。  

2. **Q：访问PHP显示502 Bad Gateway？**
   A：一般是PHP\-FPM没启动，或者Nginx配置里fastcgi\_pass的地址不对。先看php\-fpm状态，再看Nginx错误日志。  

3. **Q：WordPress安装时连不上数据库？**
   A：检查数据库主机、用户名、密码、数据库名是否正确；MySQL服务是否启动；用户有没有对应数据库的权限。  

4. **Q：上传图片/安装主题失败？**
   A：99%是权限问题。检查wp\-content目录的所有者是不是nginx，用chown改一下权限。  

5. **Q：yum安装软件包很慢怎么办？**
   A：换成国内的yum源，比如阿里云源、清华源，速度会快很多。

## 技术问题应急预案

### 网络问题

- **教室网络差**：提前和学校确认网络情况；自己准备个4G/5G热点备用；把需要下载的软件包提前下载到U盘

- **阿里云访问慢**：提前把MySQL源、PHP源、WordPress安装包下载到本地，用局域网分发给学生

- **学生手机没网**：让学生互相分享热点，或者教师开热点备用

### 服务器问题

- **学生申请不了免费服务器**：①提前准备几台备用服务器，几个人共用一台；②让有多个手机号的同学帮忙申请；③实在不行就用本地虚拟机演示

- **学生把系统搞坏了**：教他们怎么重置系统盘，几分钟就能恢复，就当练手了

- **某个地域没资源了**：建议换个地域申请，比如北京、上海、杭州，不影响学习

- **实名认证通不过**：让学生用家长的账号，或者和同学共用

### 软件安装问题

- **yum源速度慢**：提前教学生配置阿里云yum源，速度快很多

- **某个包安装失败**：准备好替代方案，比如MySQL装不了就用MariaDB，基本兼容

- **MySQL安装太费时间**：可以考虑用MariaDB代替，CentOS 7默认源里就有，安装更快

- **WordPress下载慢**：提前下载好安装包，放在本地HTTP服务器上，让学生从本地下载



### 安全注意事项

1. 服务器是公网的，密码要设置复杂一些，不要用123456这种弱密码，分分钟被黑客扫到

2. 不要在服务器上存储敏感数据、个人隐私信息

3. 禁止用服务器做任何违法违规的事情（搭建翻墙、挖矿、攻击、色情网站等）

4. 操作要谨慎，特别是rm命令，删错了找不回来

5. 课程结束后务必释放服务器或申请退费，避免持续扣费产生纠纷

6. 教师不要帮学生保管阿里云账号密码，让学生自己管理，责任自负

7. 提醒学生注意保护个人信息，实名认证的账号不要借给陌生人用
