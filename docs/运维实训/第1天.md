### 核心理论知识点

**1.2 什么是云计算与ECS**

> **理论知识点**：

- **云计算**：通过网络按需提供可配置的计算资源（网络、服务器、存储、应用和服务）的模式
- **IaaS**（基础设施即服务）：提供虚拟化的计算资源，如ECS
- **ECS**（Elastic Compute Service）：弹性计算服务，一台ECS实例等同于一台虚拟服务器，内含CPU、内存、操作系统、网络配置、磁盘等基础组件
- **与传统物理服务器的对比**：
  - 传统服务器：采购周期长、硬件维护成本高、资源利用率低
  - 云服务器：分钟级开通、按需付费、弹性伸缩、免运维硬件



### 云计算三种服务模式

| 模式     | 全称           | 运维工作                | 例子                 |
| -------- | -------------- | ----------------------- | -------------------- |
| **IaaS** | 基础设施即服务 | 全部自己（装系统+部署） | 阿里云ECS、腾讯云CVM |
| **PaaS** | 平台即服务     | 少量（部署代码）        | 阿里云RDS、新浪SAE   |
| **SaaS** | 软件即服务     | 无（用户做什么）        | 钉钉、飞书、邮箱     |



🔥 三者对比总结

```
控制权：IaaS > PaaS > SaaS

复杂度：IaaS > PaaS > SaaS

自由度：IaaS > PaaS > SaaS
```

> 越往IaaS走，越接近运维工程师
> 越往SaaS走，越接近普通用户

1. **连不上？先ping一下**

# 三、Linux介绍

## Linux操作系统介绍

### **📌Linux是什么**

​	Linux 并不是特指某一个孤立的软件，它是一个**基于内核的、开源的、多用户多任务的类 Unix 操作系统**。

- 多用户系统
- 开源系统
- 服务器主流系统



**📌Linux 内核 vs 发行版**

- **Linux 内核（Kernel）**：由 Linux 之父林纳斯·托瓦兹（Linus Torvalds）在 1991 年写出的核心代码。它负责直接驱动硬件、管理内存和 CPU，相当于汽车的“发动机”。
- **Linux 发行版（Distribution）**：各大厂商或开源社区，拿了 Linux 的“发动机”，在外面套上壳（各种管理工具、软件源、甚至图形界面），打包成一个可以直接安装的系统。

**📌Linux三大主流阵营**

- **Red Hat系列**：RHEL、**CentOS**、Alibaba Cloud Linux（企业最爱）
- **Debian系列**：Debian、Ubuntu（界面友好，多用于 AI 和区块链项目）
- **国产操作系统**：如欧拉（openEuler）、龙蜥（Anolis OS），都是基于 Linux 内核开发的。

**Linux与Windows对比**

| 对比项           | Linux        | Windows            |
| ---------------- | ------------ | ------------------ |
| 界面与资源消耗   | 默认无       | 多                 |
| 稳定性和运行周期 | 极度稳定     | 频繁更新，重启     |
| 费用             | 开源免费     | 商业收费           |
| 安全性           | 高           | 相对较低           |
| 权限管理机制     | 极其严苛     | 默认管理员权限过高 |
| 命令行           | 强大（核心） | 较弱               |
| 服务器市场       | 主导         | 较少               |

🎯 **重点**：

> Windows是给人用的， Linux是给服务器用的
>
> 运维工程师80%的工作 = 在黑窗口敲命令

**Linux文件系统结构**

```
/           # 根目录
├── bin/    # 基础命令
├── boot/   # 启动文件
├── dev/    # 设备文件
├── etc/    # 配置文件
├── home/   # 用户目录
├── root/   # root用户目录
├── tmp/    # 临时文件
├── usr/    # 应用程序
└── var/    # 可变数据（日志等）
```

- `/etc`：所有服务的配置中心（修改后需重启服务生效）  
- `/var/log`：运维故障排查第一现场



![image-20260707091843013](https://img.aa520.cn/test/image-20260707091843013.png)



**记忆技巧：**不要死记硬背，理解每个目录的设计目的。比如etc是Editable Text Configuration的缩写，就是放配置文件的；var是variable的缩写，放经常变化的文件。理解了就好记了。



绝对路径 vs 相对路径

- **绝对路径**：从根目录\(/\)开始的完整路径，比如 /etc/nginx/nginx\.conf

- **相对路径**：相对于当前位置的路径，比如 \./nginx\.conf 或 \.\./etc/passwd

- `.` 代表当前目录

- `..` 代表上一级目录

- `~` 代表当前用户的家目录

- `-` 代表上一次所在的目录（cd \- 可以在两个目录间来回切换）



## Linux的设计哲学

- **哲学思维**：Linux 认为“**一切皆文件**”，哪怕是一块网卡、一个正在运行的进程，在系统里都有一个对应的文本文件。**学会了改文本文件，你就学会了 Linux 运维。**

1. **一切皆文件**
   
   - Linux “一切皆文件” 的设计哲学，没有 C 盘 D 盘的概念。
   
   - 硬件设备、网络套接字、管道等都抽象成文件，用统一的接口操作
   
   - 好处：简化设计，统一编程接口，一切都可以用读写文件的方式操作
   
   - 举例：/dev/sda是硬盘文件，/dev/null是黑洞文件
   
2. **由众多小程序组成，每个程序只做一件事**
   - 每个命令就是一个小程序，功能单一但做到极致

   - 通过管道（\|）把多个程序组合起来完成复杂任务

   - 对比Windows的"大而全"软件，Linux是"小而美"的组合

   

> Windows是给人用的， Linux是给服务器用的
>
> 运维工程师80%的工作 = 在黑窗口敲命令
>
> 就像乐高积木，每个小块功能简单，但能拼出各种东西

### 一、文件与目录操作命令（核心）

**1.1 pwd —— 显示当前路径**


> **理论**：Linux采用树形目录结构，pwd（print working directory）显示当前所在位置。

**1.2 ls —— 列出目录内容**

```bash
ls              # 列出当前目录
ls -l           # 输出详解:长格式显示（权限、大小、时间）
ls -a           # 显示所有文件（包括隐藏文件）
ls -la          # 组合使用
ls /home        # 列出指定目录
```

> **理论**：Linux中`.`开头的文件为隐藏文件，`-l`参数显示详细信息。

**1.3 cd —— 切换目录**

```bash
cd /home        # 进入/home目录
cd ..           # 返回上一级目录
cd ~            # 回到用户家目录
cd /            # 回到根目录
cd -            # 回到上一个目录
```

> **理论**：绝对路径以`/`开头，相对路径从当前目录开始。
>
> **小技巧：**按Tab键可以自动补全命令和路径，输入前几个字母按Tab就行，能省很多事，还能避免打错字。

**1.4 mkdir —— 创建目录**

```bash
mkdir myfolder          # 创建单级目录
mkdir -p a/b/c          # 创建多级目录（父目录不存在时自动创建）
```

> **理论**：`-p`参数（parent）可递归创建多级目录。

5\. rmdir \- 删除空目录

注意：rmdir只能删空目录，要删非空目录用 rm \-r

**1.5 touch —— 创建空文件**

```bash
touch index.html        # 创建空文件
touch file1.txt file2.txt  # 创建多个文件
```

> **运维场景**：快速创建占位文件、修改文件时间戳，是日常运维的高频轻量操作。

**1.6 cp —— 复制文件或目录**

```bash
cp source.txt dest.txt      # 复制文件
cp -r source_dir dest_dir   # 递归复制目录
cp -i file1 file2           # 覆盖前提示确认
```

> **理论**：`-r`（recursive）参数用于复制目录。

**1.7 mv —— 移动或重命名**

```bash
mv oldname.txt newname.txt  # 重命名
mv file.txt /home/          # 移动文件
```

> **理论**：mv的本质是修改文件路径，同一目录下是重命名，不同目录间是移动。

**1.8 rm —— 删除文件或目录（危险命令！）**

```bash
rm file.txt                 # 删除文件
rm -r myfolder              # 递归删除目录
rm -rf myfolder             # 强制递归删除（⚠️ 危险！）
```

> ⚠️ **安全警告：Linux 最危险命令，无回收站机制**
>
> - `rm`删除的文件无法恢复，执行前务必用`ls`确认路径和文件名
> - **绝对禁止执行 `rm -rf /`**，会清空整个系统文件，造成不可逆灾难
> - 生产环境删除重要文件前必须先备份，新手推荐先用`rm -i`确认
> - 线上操作遵循 “先备份、再操作、慢执行、勤确认” 原则

10\. find \- 查找文件

**功能**：在指定目录下按条件查找文件

```bash
find /etc -name "*.conf"       # 在/etc目录下查找所有以.conf结尾的文件
find /root -name "*.html"      # 在/root下查找所有.html文件
```

> **参数说明**：
>
> - `-name`：按文件名查找，支持`*`通配符
> - 运维场景：服务器上定位配置文件、日志文件、异常脚本，是排查问题的基础工具。

**11 cat —— 查看文件内容**

```bash
cat /etc/hosts             # 查看文件全部内容
cat -n /etc/passwd         # 显示内容的同时显示行号
```

注意：cat适合看小文件，大文件用more或less

12\. more / less \- 分页查看文件

推荐用less，功能更强大，"less is more"（少即是多，Unix哲学的体现）

```bash
more /etc/services         # 分页查看，按空格翻下一页，按q退出（只能向下翻）
less /etc/services         # 增强版分页查看
```

> - `less`常用操作：空格下一页、b 上一页、q 退出、`/关键词`搜索、n 跳到下一个匹配项

13\. head / tail \- 查看文件开头/结尾

```bash
head /etc/passwd           # 默认查看前10行
head -n 5 /etc/passwd      # 指定查看前5行

tail /etc/passwd           # 默认查看后10行
tail -n 5 /etc/passwd      # 指定查看后5行
tail -f /var/log/messages  # 实时追踪文件末尾新增内容（监控日志神器）
```

> **运维场景**：`tail -f`是排查线上问题的高频命令，用于实时观察日志输出，按`Ctrl+C`退出。

14\. grep \- 在文件中搜索关键词（文本搜索神器）

```bash
grep "bash" /etc/passwd        # 搜索文件中包含bash的行
```

> **运维场景**：在日志、配置文件中快速定位错误信息、指定记录，是文本排查的核心命令，常与管道符搭配使用。

管道与重定向（了解）

- **管道 \|**：把前一个命令的输出作为后一个命令的输入    

```bash
# 查找nginx相关的进程
ps aux | grep nginx
```

> **设计思想**：对应 Linux“每个程序只做一件事，通过组合完成复杂任务” 的设计哲学，是运维自动化的基础语法。

- **重定向 \>**：把命令输出写入文件（覆盖）    

```bash
echo "Linux Course" > hello.txt
```

- **追加重定向 \>\>**：追加到文件末尾    

```bash
# 追加重定向：将内容追加到文件末尾，不覆盖原有内容
echo "new line" >> hello.txt
```

> **运维场景**：保存命令执行结果、生成配置文件、记录日志等。
