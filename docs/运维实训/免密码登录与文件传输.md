# SSH 免密登录与文件传输学习笔记

## 📌 场景需求
在 **GitLab CI/CD 自动化发布** 流水线中，Runner 需要将构建好的制品（如编译后的代码、Docker 镜像）远程部署到生产服务器。由于流水线运行在无人值守的非交互式环境中，无法手动输入密码，因此必须配置 **SSH 免密登录**，以实现自动化、安全、可重复的部署流程。

---

## 一、SSH 免密登录的原理
SSH 采用**公钥认证**机制：
- 客户端本地生成一对密钥：**私钥**（保密）和**公钥**（可公开）。
- 将公钥预先放置在远程服务器的 `~/.ssh/authorized_keys` 文件中。
- 登录时，客户端使用私钥签名发送请求，服务端用公钥验证签名，若匹配则允许登录，无需密码。

这种方式不仅用于 `ssh`，也天然支持 `scp`、`rsync` 等基于 SSH 协议的工具，是实现自动化部署的基石。

---

## 二、核心实战：三步实现免密登录

### 第一步：在本地机器上生成密钥对
使用 `ssh-keygen` 命令生成高强度的 RSA 密钥对（推荐 4096 位）。

```bash
ssh-keygen -t rsa -b 4096
```

**📥 终端交互示例：**


> **💡 重要提示**：
>
> - 默认路径为 `~/.ssh/id_rsa`（私钥）和 `~/.ssh/id_rsa.pub`（公钥）。
> - **Passphrase（密钥密码）务必留空**，否则自动化脚本执行时仍需手动输入，违背免密初衷。
> - 若已有密钥对，可跳过此步，直接使用现有公钥。

------

### 第二步：将公钥复制到远程服务器

#### 🔧 方法一：使用 `ssh-copy-id`（最推荐）

`ssh-copy-id` 是 OpenSSH 提供的专用工具，自动完成公钥追加和权限修复。

**命令格式：**

bash

```
ssh-copy-id [-i 公钥文件] [user@]hostname
```



**示例（使用默认公钥）：**

bash

```
ssh-copy-id root@remote-host
```



**若指定非默认公钥：**

bash

```
ssh-copy-id -i ~/.ssh/mykey.pub root@remote-host
```



**📥 执行过程：**

text

```
root@remote-host's password: [输入远程服务器密码]
Number of key(s) added: 1
Now try logging into the machine, with:   "ssh 'root@remote-host'"
and check to make sure that only the keys you expected were added.
```



> **✨ `ssh-copy-id` 的优势**：
>
> - 自动检查远程 `~/.ssh` 目录是否存在，若不存在则创建。
> - 自动纠正目录和文件权限（`~/.ssh` 为 700，`authorized_keys` 为 600），避免因权限过宽导致 SSH 拒绝免密。
> - 如果远程已有该公钥，不会重复添加。

#### 📋 方法二：手动复制（备选）

若无法使用 `ssh-copy-id`（如 Windows 原生环境），可手动操作：

bash

```
# 查看本地公钥内容
cat ~/.ssh/id_rsa.pub

# 登录远程服务器，将公钥内容追加到 authorized_keys
mkdir -p ~/.ssh
echo "公钥内容" >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

### 第三步：验证免密登录与文件传输

#### 1. 验证 SSH 登录

bash

```
ssh root@remote-host
```



如果直接进入远程 Shell 而无需密码，说明配置成功。

#### 2. 验证 SCP 文件传输

bash

```
scp ./dist.tar.gz root@remote-host:/var/www/html/
```



传输过程不再提示密码，可直接用于自动化脚本。

#### 3. 验证 Rsync（常用作增量同步）

bash

```
rsync -avz -e ssh ./build/ root@remote-host:/app/
```

## 三、常见问题排查与避坑指南

### 🔒 权限问题导致免密失败

即使公钥正确，若远程服务器目录权限不符合 SSH 安全严格模式（`StrictModes yes`），仍会要求密码。
**修复方法**（登录远程服务器执行）：

bash

```
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
# 确保属主正确（若为普通用户，则 chown 对应用户）
chown -R $USER:$USER ~/.ssh
```



### 🔍 检查 SSH 服务端日志

若仍失败，查看服务端日志（通常为 `/var/log/auth.log` 或 `/var/log/secure`）：

bash

```
tail -f /var/log/auth.log | grep sshd
```



常见错误如 `Authentication refused: bad ownership or modes` 均指向权限问题。

### 🧹 清理多余的 authorized_keys 记录

如果 `authorized_keys` 中存在多个公钥，可手动删除无效行，避免冲突。

### 🧪 使用 `ssh -v` 调试

bash

```
ssh -v root@remote-host
```



详细输出会显示认证过程，帮助定位问题。

## 四、CI/CD 自动化落地实战（以 GitLab 为例）

在实际的 GitLab CI/CD 流水线中，Runner 通常是临时容器，无法提前预置密钥。**标准做法**是使用 CI 变量动态注入私钥。

### 步骤 1：将私钥添加到 GitLab CI/CD 变量

- 在本地生成密钥对后（**公钥已部署到目标服务器**），将**私钥文件内容**（`id_rsa`）完整复制。
- 进入 GitLab 项目 → **Settings** → **CI/CD** → **Variables** → 添加变量：
  - **Key**：`SSH_PRIVATE_KEY`
  - **Value**：粘贴私钥内容（包括 `-----BEGIN OPENSSH PRIVATE KEY-----` 等所有行）
  - **Type**：选择 `File`（推荐）或 `Variable`
  - **Protect** / **Mask** 根据安全策略设置。

### 步骤 2：在 `.gitlab-ci.yml` 中注入密钥

yaml

```
before_script:
  # 创建 .ssh 目录并设置权限
  - mkdir -p ~/.ssh
  - chmod 700 ~/.ssh

  # 将私钥变量写入文件（若 Type=File，则直接使用 $SSH_PRIVATE_KEY 路径）
  - echo "$SSH_PRIVATE_KEY" > ~/.ssh/id_rsa
  - chmod 600 ~/.ssh/id_rsa

  # 添加目标主机到 known_hosts（避免交互确认）
  - ssh-keyscan -H remote-host >> ~/.ssh/known_hosts
  - chmod 644 ~/.ssh/known_hosts

deploy:
  script:
    - scp ./dist.tar.gz root@remote-host:/var/www/html/
    - ssh root@remote-host "cd /var/www/html && tar -xzf dist.tar.gz"
```



> **⚠️ 安全提醒**：
>
> - 私钥变量务必设置为 **Masked**，防止在日志中泄露。
> - 定期轮换密钥对，并限制目标服务器上的用户权限（如使用专用部署账号，而非 root）。
> - 使用 `ssh-keyscan` 预先添加主机指纹，可避免首次连接时的确认提示。

## 五、扩展技巧与最佳实践

### 🛠️ 使用 `ssh-agent` 管理多个密钥

当本地有多个密钥时，可使用 `ssh-agent` 缓存私钥：

bash

```
eval $(ssh-agent -s)
ssh-add ~/.ssh/id_rsa
```



之后所有 SSH 连接自动使用已添加的密钥。

### 📦 结合 Ansible 批量管理

在 Ansible 的 inventory 文件中配置 `ansible_ssh_private_key_file`，即可对大批量服务器执行免密操作。

### 🌐 跨平台兼容性

- **Windows**：可使用 Git Bash、WSL 或 PowerShell（需安装 OpenSSH 客户端）执行上述命令。
- **macOS/Linux**：原生支持。
